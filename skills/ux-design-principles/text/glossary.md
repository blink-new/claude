# UX Design Principles — Full Glossary

A comprehensive vocabulary reference for engineers, designers, and product managers.
Organized alphabetically. Use this to speak precisely about design with AI assistants and teammates.

---

## A

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Accent color** | A contrasting color used sparingly to highlight key elements | Blue price tag on a neutral card |
| **Active state** | Visual feedback during the moment a click/tap is occurring | Button gets darker + slightly shifts down |
| **Affordance** | The potential action an element supports | A button affords clicking |
| **Animation** | Motion over time that changes element appearance or position | Chip sliding up on copy action |
| **Aspect ratio** | Width-to-height proportion of an element | 16:9 for video, 1:1 for avatar |

## B

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Backdrop filter** | CSS filter applied to content behind an element | Frosted glass modal backdrop |
| **Body text** | Paragraph-length reading text | 16px regular, 160% line-height |
| **Border radius** | Rounding applied to corners | 8px for cards, 9999px for pill buttons |
| **Box shadow** | CSS property for outer (or inner) shadows | Subtle depth on cards |
| **Breakpoint** | Pixel width where layout changes for responsiveness | 768px = tablet, 1024px = desktop |

## C

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Call to action (CTA)** | A button or link prompting a specific user action | "Start for free", "Get started" |
| **Card** | A contained UI unit grouping related content | Product listing, blog post preview |
| **Color ramp / scale** | A spectrum of tints and shades of one hue (50-900) | Tailwind's blue-50 through blue-900 |
| **Confetti** | Celebration particle animation for major milestones | First project published |
| **Contrast** | Degree of difference between two elements (size, color, weight) | Large bold title vs small muted metadata |
| **Contrast ratio** | WCAG accessibility metric for text legibility | Min 4.5:1 for body text (AA) |
| **Copy button** | Button that copies content to clipboard | "Copy" on a code block |
| **Cursor** | The visual indicator of the pointer position | `cursor: pointer` for clickable elements |

## D

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Dark mode** | UI variant designed for low-light environments | Dark background, light text |
| **Dark surface** | A dark-colored background layer in dark mode | Card at #1A1A1A on background #0A0A0A |
| **De-emphasis** | Reducing size/weight/color to lower hierarchy | Small gray metadata below title |
| **Default state** | The resting appearance of an element | Button before any interaction |
| **Depth** | Perceived z-level of an element in 3D space | Modal appears above page content |
| **Disabled state** | Non-interactive appearance — muted, no hover | opacity-50, cursor-not-allowed |
| **Display text** | Very large text for headlines (56px+) | Hero section header |
| **Drop shadow** | CSS filter shadow that follows element contour | Shadow respects rounded corners |

## E

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Easing** | The acceleration curve of an animation | ease-out for entering elements |
| **Elevation** | Perceived z-height; higher = stronger shadow or lighter surface | Modal > popover > card |
| **Empty state** | The UI shown when a list/view has no content | "No projects yet. Create one →" |
| **Enter animation** | Motion played when an element appears | Fade in + slide up |
| **Error state** | Communicates invalid input or failed operation | Red border + helper text |
| **Exit animation** | Motion played when an element disappears | Fade out + slide down |

## F

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Feedback** | Visual/motion response confirming a user action | "Copied!" chip on copy click |
| **Focus ring** | Visible outline when element has keyboard focus | Blue ring on focused input |
| **Focus state** | Input border highlight when user clicks in | Blue border + glow |
| **Focal point** | The single element that draws the eye first | Large product image |
| **Font** | A specific weight/style within a typeface | Inter Bold 700 |
| **Font weight** | Thickness of letterforms (100-900) | 400 = regular, 700 = bold |
| **4-point grid** | Design system where all spacing is a multiple of 4px | 8, 16, 24, 32, 48px |
| **Frosted glass** | Blurred semi-transparent surface effect | backdrop-filter: blur(12px) |

## G

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Ghost button** | Button with transparent background; border or text only | Secondary CTA beside primary |
| **Gradient** | Smooth transition between two or more colors | Linear gradient overlay on hero image |
| **Grid** | System of columns/rows for aligning content | 12-column layout grid |
| **Gutter** | Space between columns in a grid | 24px between columns |

## H

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Hierarchy (visual)** | Order elements are perceived in, from most to least important | Large title > medium body > small metadata |
| **HEX** | 6-digit color code (#2563EB) | Most common in CSS |
| **Hover state** | Visual change when cursor enters element bounds | Button lightens/darkens |
| **HSL** | Hue-Saturation-Lightness color format | hsl(221, 83%, 53%) |

## I

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Icon** | A small symbolic image representing an action or concept | Magnifying glass = search |
| **Icon-only button** | Button with only an icon; needs tooltip | Trash icon to delete |
| **Information density** | Amount of content packed into a given space | Dashboard vs landing page |
| **Inner shadow (inset)** | Shadow appearing inside an element | Pressed button effect |

## K

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Keyframe** | A defined state at a specific point in an animation | 0% { opacity: 0 } to 100% { opacity: 1 } |

## L

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Landing page** | Marketing/sales page for a product | Wide font range, dramatic hierarchy |
| **Leading (line-height)** | Vertical space between lines of text | 24px for 16px body text |
| **Letter-spacing (tracking)** | Horizontal spacing between characters | -2% on large headlines |
| **Linear gradient** | Gradient along a straight line | Top-transparent to bottom-black on images |
| **Loading state** | Indicates async operation in progress | Spinner on button |

## M

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **mask-image** | CSS property clipping visibility with a gradient | Makes blur apply only to bottom half |
| **Micro-interaction** | Small animation triggered by a user action | Chip slides up on copy click |
| **Modal** | A dialog that appears above page content | Confirmation dialog |
| **Modal backdrop** | Dark/blurred overlay behind modal | rgba(0,0,0,0.5) or blur |
| **Muted color** | A color with reduced saturation | Soft green chip background in dark mode |

## N

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Negative space (whitespace)** | Empty space between/around elements | 32px between sections |
| **Neutral** | Black, white, and grays | Foundation of most UI |

## O

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Opacity** | How transparent an element is (0=invisible, 1=opaque) | Disabled state: opacity-50 |
| **Optimistic UI** | Showing success state before server confirms | Like count increments immediately |
| **Overlay** | A layer positioned on top of content | Gradient over hero image |

## P

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Padding** | Internal space between content and element edge | px-4 py-2 on a button |
| **Parallax** | Background moves slower than foreground on scroll | Depth effect on marketing pages |
| **pill button** | Button with fully-rounded corners | Tags, chips, filter buttons |
| **Primary CTA** | The main call-to-action; one per section | "Get started" button |
| **Progressive blur** | Blur intensity increasing across a gradient | Modern card overlay technique |
| **Proximity** | Grouping elements visually by placing them close | Icon + label vs. label + gap + icon |

## R

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Radial gradient** | Gradient radiating from a center point | Vignette on photo backgrounds |
| **Responsive** | Layout adapts to different screen sizes | Stack vertical on mobile |

## S

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Sans-serif** | Font without decorative strokes | Inter, Geist, DM Sans |
| **Saturation** | Color intensity (high=vivid, low=muted) | Chip background muted in dark mode |
| **Scanning pattern** | Path user's eye takes across a page | F-pattern on content, Z on marketing |
| **Scrim** | Translucent overlay making content more readable | Gradient scrim on hero image |
| **Secondary CTA** | Supporting action alongside primary | "Learn more" ghost button |
| **Semantic color** | Color that carries convention-based meaning | Red=error, green=success |
| **Serif** | Font with decorative strokes | Georgia, Playfair Display |
| **Shadow** | Visual cue creating depth/elevation | box-shadow on cards and modals |
| **Signifier** | Visual cue communicating an element's affordance | Hover state = "this is clickable" |
| **Skeleton screen** | Layout placeholder shown while content loads | Gray shimmer blocks |
| **Spacing** | Distance between/around elements | 4pt grid multiples |
| **Spring animation** | Physics-based bouncy animation | Toggle switch settles into position |
| **Stagger** | Sequential delays on a list of animating elements | Nav items animate in one by one |
| **State** | One of multiple visual/behavioral modes | Default, hover, active, disabled |
| **Surface elevation** | Using lighter backgrounds in dark mode for depth | Card (#1A1A) on background (#0A0A) |

## T

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Tactile button** | Button using shadows to simulate physical depth | Raised/pressed 3D feel |
| **Toast** | Temporary auto-dismissing notification | "Saved successfully!" |
| **Toggle** | Binary control switching between two states | On/off switch |
| **Tooltip** | Contextual info appearing on hover | Hover an icon-only button |
| **Tracking (letter-spacing)** | Horizontal space between characters | Negative on large headings |
| **Transition** | CSS-based animation between two states | color 150ms ease |
| **Trigger** | User action initiating a micro-interaction | Click, hover, focus, scroll |
| **Type scale** | Defined set of font sizes in a design system | 12, 14, 16, 20, 24, 32, 48px |
| **Typeface** | A font family (all weights and styles) | Inter |

## V

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Vignette** | Darkening toward image edges via radial gradient | Photo aesthetic effect |
| **Visual hierarchy** | Order in which elements are perceived | Large title first, metadata last |
| **Visual weight** | How much attention an element demands | Large bold colored title = high weight |

## W

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **WCAG** | Web Content Accessibility Guidelines — contrast/size standards | 4.5:1 ratio for body text |
| **Whitespace** | Empty space in a design; creates breathing room | 32px between sections |

## Z

| Term | Definition | Example in Practice |
|------|-----------|-------------------|
| **Z-index** | CSS stacking order | Modal (z-50) above overlay (z-40) above page (z-0) |
