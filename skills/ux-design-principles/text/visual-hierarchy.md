# Visual Hierarchy

## What Is Visual Hierarchy?

Visual hierarchy is the arrangement and presentation of elements so the user's eye travels in the *intended order* — from most important to least important.

> "It's this contrast — the difference between small and big, or colorful and not — that actually creates the hierarchy."

---

## The Three Tools of Hierarchy

### 1. Size
- Large = important
- Small = supporting detail
- The bigger the contrast between sizes, the stronger the hierarchy

### 2. Position
- Top = seen first
- Left = read first (in LTR languages)
- Top-right = ideal for prices and key metrics (draws the eye as outlier)

### 3. Color
- Bold, saturated color = high importance
- Muted, gray = low importance / secondary info
- **Colorful items that differ from their surroundings attract attention automatically**

---

## Card Hierarchy Example (Step-by-Step)

**Before (no hierarchy — looks like a spreadsheet):**
All text the same size, same weight, same color. Everything competes.

**After (with hierarchy):**

| Element | Treatment | Reason |
|---------|-----------|--------|
| Hero image | Full-width, top of card | Adds color, aids scanning instantly |
| Item name | Large, bold, top-left | Most important; stands out |
| Price | Large, top-right, blue | Different from everything else; eye goes here |
| Time/Date | Small, below name | Supporting detail; less important |
| Location | Small, with icon + line graphic | Communicates direction without text labels |

**Key insight:** Price should be top-right and blue *because it's different*. That difference is the hierarchy signal.

---

## Hierarchy Is Not Exact Science

> "It wouldn't be wrong, but the same ideas apply: most important things near the top, bigger and colorful is more important, images used whenever possible."

Two valid designs can achieve the same hierarchy in different ways. The *principles* apply; the *implementation* has creative latitude.

---

## Practical Rules

1. **Never make everything the same size** — hierarchy requires contrast
2. **Images always help** — they add visual weight and aid scanning
3. **Put the most important thing first** (top or top-left)
4. **Price, CTA, or key metric** gets an accent color to stand out
5. **Supporting info** (timestamps, metadata) should be small, muted, and below
6. **Icons + visual connectors** can replace verbose labels (e.g., an arrow line from A to B instead of "From: A, To: B")

---

## Information Density Consideration

| Context | Hierarchy approach |
|---------|-------------------|
| Landing pages / marketing | Wide size range, big images, dramatic contrast |
| Dashboards | Tighter size range (max ~24px heading), denser layout |
| Mobile cards | Image top, title bold, metadata small/muted |
| Data tables | Minimal visual hierarchy — consistency > contrast |

---

## Engineering Vocabulary

| Term | Definition |
|------|-----------|
| **Visual hierarchy** | Order in which the eye processes elements on screen |
| **Contrast** | The degree of difference between two design elements (size, color, weight) |
| **Visual weight** | How much attention an element demands relative to its surroundings |
| **Focal point** | The single element that draws the eye first |
| **Scanning pattern** | The path a user's eye takes across a page (F-pattern, Z-pattern) |
| **Information density** | Amount of content packed into a given space |
| **De-emphasis** | Reducing size, weight, or color to push element down the hierarchy |
| **Accent color** | A contrasting color used sparingly to highlight key elements |
